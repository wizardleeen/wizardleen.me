本文记录了我用 Claude Code（以下简称 CC）开发一个类 Vercel 轻量级云平台的完整过程，以及其中的思考与总结。

## 产品介绍

### 问题

在国内部署前端项目一直很麻烦。以阿里云为例，搭建网站需要配置 OSS、CDN、证书，每次部署都要手动上传并刷新缓存。而 Vercel、CloudFlare 等海外平台没有国内 CDN 节点，速度慢，且时常被墙。

### 解决方案

我基于阿里云构建了一个类 Vercel 的部署平台——**秒部**，实现 git push 一键部署。

项目地址：https://miaobu.metavm.tech

### 核心功能

- git push 一键部署
- 支持前端、Node 及 Python 后端
- 自动签发 HTTPS 证书
- 自定义域名绑定
- CDN 全球加速
- 快速回滚

目前已上线，接入多个站点。

## 开发过程

秒部完全由 CC 开发，没有人工编写一行代码。

### 原型编写

第一步是让 CC 编写基础原型，Prompt 如下：

```markdown
Build a lightweight, Vercel-like deployment platform targeting the China market, powered by Alibaba Cloud.
Core requirements:

* Deploy frontend projects with a single git push
* GitHub OAuth for user login
* Alibaba Cloud OSS + ESA for website hosting and delivery

Tech stack:

* Python backend
* Modern, clean architecture
```

### 环境搭建

代码编写完成后，我让 CC 在本地用 Docker 启动前端、后端、数据库等容器。由于容器操作无需 sudo 权限，CC 可以直接管理整个环境。

本地环境就绪后，接下来便是云环境集成。秒部需要调用阿里云 API 来操作 OSS 和 CDN，而这一环节容易出错——API 用错、密钥配错、Bucket ID 不对等等。手动调试很耗时，于是我直接给 CC 一个临时的阿里云 Access Key，让它自行操作和调试云资源。

### 问题排查

CC 能直接与运行环境交互时，排查能力极强，因为它能快速收集各环节的信息——这恰恰是人类开发者最头疼的部分。例如一次部署失败涉及 GitHub、秒部服务和阿里云三方，人工逐一排查费时费力，而 CC 快速收集信息后很快定位了问题。

### 持续迭代

原型可用后，我继续让 CC 实现更多功能：Python/Node 后端支持、版本回滚、UI 重设计等。工作流如下：

* 开启会话，提出需求
* Review CC 的方案，进入编码
* 部署本地容器，调试
* 编写 commit 信息，提交代码
* 迁移数据，部署上线
* 整理本次会话中的新知识并保存
* 关闭会话

CC 大部分时候都能独立完成调试，但偶尔会卡壳——可能是环境配置差异或知识盲区。此时需要你主动提供线索来帮它突破。

### 架构把控

用 CC 写代码很容易养成惰性：只管提需求，不关心实现。但这行不通——至少现在还不行。

AI 写代码有**短视倾向**：关注眼前需求或当前 Bug，倾向选择最简方案，往往导致糟糕设计。比如 CC 在 API 接口中用轮询检查服务状态——这是典型的反模式。它之所以这样做，是因为秒部当时没有 Worker 模块，而在"新建模块"和"简单方案先顶上"之间，CC 选了后者。

随着项目复杂度增长，架构把控越发关键。开发者必须理解并主导架构演进，**绝不能让项目变成黑盒**。

### 多线并行

用 CC 开发时你会发现自己的时间反而富余了——大部分时间都在等 CC 完成任务。因此我经常同时推进多个项目：一个会话在实现新需求，另一个在排查并发问题，而我自己在设计新特性。

### AI 与开发者的分工

CC 优势显著，短板也明显。

**优势：** 在代码编写、文档撰写、Commit 信息等具体任务上，AI 表现优于 95% 的开发者。它能不厌其烦地处理 edge case、写注释、写单测——这些人类最容易偷懒的地方。

**短板：** 缺乏长远规划和系统性架构思维，仍需人类指导。不过我相信，架构设计终将被 AI 攻克，届时开发工作可完全由 AI 承担。

这对不同角色的影响截然不同：**创造者**会获得革命性工具，过去受限于资源的想法现在可以快速落地；而纯粹的**需求执行者**将面临巨大冲击，因为这部分工作很可能被完全取代。

## 总结

五天时间，从零到上线一个类 Vercel 平台——这在以前几乎不可想象。我们正在经历一次行业的巨大变革，还没有上车的小伙伴，抓紧了！
